# 02. Зашитый текст обходит C# setter

## Суть

Harmony-патч на `TextMesh.text` setter перехватывает только **программные** присваивания из C#-кода. Текст, зашитый в префабы и сцены, устанавливается движком Unity при десериализации **в обход управляемого setter'а**. Патч его не видит.

## Постановка проблемы

Текст кнопок главного меню («New Game», «Options», «Quit») хранится в `.unity`-сценах и `.prefab`-файлах как сериализованное поле `m_Text` компонента TextMesh. При загрузке сцены Unity восстанавливает это поле через native-код десериализации, минуя C# property accessor. Harmony-prefix на setter не срабатывает.

Симптом: Harmony-патч установлен и подтверждён срабатыванием (`[DIAG] ПАТЧ TextMesh.text СРАБОТАЛ`), но зашитые подписи остаются на языке оригинала.

## Решение

Активный сканер сцены — MonoBehaviour, периодически перебирающий все TextMesh и применяющий перевод напрямую:

```csharp
var meshes = FindObjectsOfType<TextMesh>();
foreach (var tm in meshes)
{
    string cur = tm.text;
    if (string.IsNullOrEmpty(cur)) continue;
    string translated = Translate(cur);
    if (translated != cur)
        tm.text = translated; // программное присваивание — setter сработает,
                                // но патч увидит уже кириллицу и пропустит
}
```

Запись `tm.text = translated` вызывает setter — патч срабатывает повторно, но видит кириллицу и возвращает unchanged. Рекурсии нет.

## Практические параметры

- **Интервал сканирования:** 0.5 сек достаточно для отзывчивости без перегрузки.
- **Событие пересканирования:** при `SceneManager.sceneLoaded` (сцена загрузилась → текст десериализован → сканируем).
- **Оптимизация:** пропускать пустые TextMesh и уже переведённые (содержащие кириллицу).

## Взаимодействие с setter-патчем

Оба механизма нужны:

| Механизм | Что ловит | Что пропускает |
|----------|-----------|----------------|
| Harmony setter-патч | Динамический текст (LookUI.ShowLookText, GPButtonKeybinding.UpdateText) | Зашитый в префабах |
| Активный сканер | Зашитый в префабах | Текст, установленный между проходами сканера |

Сканер покрывает пробел setter-патча; setter-патч покрывает пробел сканера (мгновенная реакция на динамический текст). Совместное использование даёт полное покрытие.
