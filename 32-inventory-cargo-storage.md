# 32. Инвентарь, груз и хранение

Разбор систем хранения предметов у игрока и перевозки грузов. Информация получена декомпиляцией `Assembly-CSharp.dll` (Sailwind v0.38) через ILSpy. Базовый фреймворк предметов — в заметке 16.

## Кодировка слотов хранения

Единая схема (используется в `SavePrefabData.inventorySlot`, заметка 11):

| Значение `inventorySlot` | Куда положен предмет |
|:--:|----------------------|
| `< 100` | Личный слот инвентаря (`GPButtonInventorySlot.inventorySlots[slot]`). |
| `>= 100` | Грузовой носитель (`CargoCarrier.carriers[slot - 100]`, индекс = `portIndex`). |
| `-1` | Не в хранилище (в мире/в руках). |

## Личный инвентарь (`GPButtonInventorySlot`)

Статический реестр `inventorySlots[5]` — **5 личных слотов** игрока.

| Поле | Содержание |
|------|-----------|
| `slotIndex` | Индекс слота (0..4). |
| `currentItem` (`ShipItem`) | Предмет в слоте. |

### Помещение предмета (`OnItemClick`)
Предмет кладётся в слот, если **все** условия:
- слот пуст (`currentItem == null`);
- предмет куплен (`sold == true`);
- предмет **не крупный** (`!big`);
- если это бутылка (`ShipItemBottle`) — её объём `<= 10` (крупные бутыли не лезут).

> **Важно для лута:** крупный предмет (`big`), в т.ч. **ящик** (`ShipItemCrate`), **не кладётся в слот инвентаря** — его берут **в руки**, ставят **на палубу**, а ящик затем **вскрывают** (`unseal`) → содержимое попадает в **`CrateInventory`** ящика, не в личный инвентарь (подробнее — заметки 33, 45).

`InsertItem`: предмет входит в слот (`ItemRigidbody.EnterInventorySlot`), звук. `WithdrawItem`: извлечение предмета (`ExitInventorySlot`).

## Грузовой носитель (`CargoCarrier`)

«Тележка»/склад для перевозки товаров **между портами**. Статический реестр `carriers[50]` — **до 50 носителей** (индекс = `portIndex`).

| Поле | Содержание |
|------|-----------|
| `portIndex` | Индекс порта (и индекс в `carriers`). |
| `currency` | Валюта оплаты услуг носителя. |
| `transportPriceMult` | Множитель цены перевозки. |
| `storagePriceMult` | Множитель цены хранения. |
| `cargo` (`List<ShipItem>`) | Хранимые товары. |

Подписан на `Sun.OnNewDay` → `RegisterDayPassed`: каждый игровой день **`daysInStorage++`** для всех грузов (накопление платы за хранение).

### Погрузка (`InsertItem`)
- **Нельзя грузить незапечатанные ящики:** `ShipItemCrate` с `amount <= 0` → `"Cannot transport unsealed crates."`
- Проверка денег; не хватает → `"Not enough money."`
- Списывается цена перевозки, предмет скрывается (`localScale = 0`), `daysInStorage = 0`.

### Цена перевозки (`GetTransportPrice`)
```
price = (121 + mass) * transportPriceMult * 0.016
       → GetBuyPriceInCurrency(currency, ...)
```
Тяжелее товар → дороже перевозка (базовая надбавка 121 к массе).

### Цена выгрузки/хранения (`GetWithdrawPrice`)
```
days = daysInStorage - 1
fee  = transportPrice * days * storagePriceMult * 2.5
fee  = min(fee, item.value)         // не больше стоимости товара
```
**Плата за хранение растёт каждый день** — чем дольше груз лежит, тем дороже его забрать (но не дороже его стоимости).

### Выгрузка (`WithdrawItem`)
Оплата цены выгрузки → предмет извлекается (сдвиг вперёд на 10 ед.), подбирается игроком, `daysInStorage = 0`. Нельзя выгружать, если уже держишь предмет.

## Связь с другими системами
- **Сохранение:** `inventorySlot` в `SavePrefabData` (заметка 11); при загрузке `>= 100` → `CargoCarrier.LoadSavedItem`, `< 100` → `GPButtonInventorySlot.InsertItem`.
- **Порча в хранилище:** `daysInStorage` влияет на порчу еды (`FoodState`, заметка 23) и на плату за хранение.
- **Ящики** (`ShipItemCrate`): должны быть запечатаны (`amount > 0`) для перевозки; печать — `CrateSealUI`.

## Практические выводы для мододела

1. **5 личных слотов**, только для мелких (`!big`) купленных предметов; бутылки — объёмом ≤ 10.
2. **До 50 грузовых носителей** (по `portIndex`); слот предмета `>= 100` = `portIndex + 100`.
3. **Перевозка** стоит `(121 + mass) * transportPriceMult * 0.016` в валюте порта.
4. **Хранение дорожает ежедневно** (`transportPrice * (days-1) * storagePriceMult * 2.5`, кап = стоимость товара) — стимул не держать груз долго.
5. **Незапечатанные ящики** нельзя перевозить (нужна печать, `amount > 0`).
6. `daysInStorage` — общий счётчик дней в хранилище (влияет на порчу и плату); сбрасывается в 0 при погрузке/выгрузке.
