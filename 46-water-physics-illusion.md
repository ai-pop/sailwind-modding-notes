# 46. Физика воды: реальность vs иллюзия

## Краткий ответ

**Вода в Sailwind — визуальная.** Настоящей физики воды (гидродинамики, сопротивления, выталкивания) нет ни для кого. Но реализация для корабля и игрока **принципиально разная**, и именно это создаёт диссонанс: корабль ведёт себя «физически правильно», а игрок — как левитирующий объект.

## Архитектура системы «воды»

### 1. Океан — визуальный меш на основе FFT

Класс `Ocean` (форк Crest) генерирует меш воды через быстрое преобразование Фурье. Это **чисто визуальная + сэмплирующая** система:

| Компонент | Что делает |
|-----------|------------|
| `vertices[]` / `normals[]` | Визуальная поверхность воды (меш) |
| `GetWaterHeightAtLocation2(x, z)` | Возвращает высоту **визуальной** поверхности в точке |
| `GetChoppyAtLocation2(x, z)` | Горизонтальное смещение волны (choppy) |
| `canCheckBuoyancyNow[0]` | Байт-гейт: `== 1` когда данные волн готовы к запросу |

**Ключевое:** `Ocean` не создаёт физического тела воды. Это «декорация с API высоты». Никакого объёма, плотности, вязкости или гидродинамики.

### 2. Корабль — иллюзия физики через `Rigidbody` + `Buoyancy`

Корабль использует **настоящий `Rigidbody`** (Unity-физику) + компонент `Buoyancy`. Это классическая blob-модель плавучести:

```
Для каждого «блоба» (точки на сетке BoxCollider):
    worldPos = TransformPoint(blobLocalPos)
    waterY = Ocean.GetWaterHeightAtLocation2(worldPos.x - choppy, worldPos.z)
    depth = worldPos.y - waterY          // глубина погружения
    force = magnitude * depth            // выталкивающая сила
    force += dampCoeff * velocity.y      // демпфирование качки
    rigidbody.AddForceAtPosition(-up * force, worldPos)
```

**Почему корабль выглядит «физически правильно»:**
- Сила прикладывается в точках погружения → возникает момент силы → корабль кренится/дифферентует реалистично
- Центр масс занижен (`CenterOfMassOffset = -1`) → самовыравнивание
- Демпфирование (`dampCoeff = 0.1`) → гашение колебаний
- Интерполяция (`interpolation = 3`) → плавность
- Ветер и choppy добавляют случайные силы

**Это НЕ физика воды — это физика твёрдого тела (Rigidbody) с обратной связью от визуальной поверхности.** Но выглядит убедительно.

### 3. Игрок — голое позиционирование без физики

Игрок использует `CharacterController` (НЕ Rigidbody) + `PlayerSwimming`. **Никакой физики:**

```csharp
// PlayerSwimming.LateUpdate()
waterHeight = Mathf.Lerp(waterHeight, UnderwaterEffect.cameraWaterHeight, deltaTime * lerp);

// Определение состояния (просто сравнение координат):
swimming = (transform.position.y <= waterHeight + swimHeight);
inShallowWater = (transform.position.y <= waterHeight + shallowWaterHeight);
swimmingOnSurface = (transform.position.y >= waterHeight + surfaceSwimHeight);

// «Покачивание на волнах» — НЕ физика, а телепортация:
if (swimming) {
    charController.Move(Vector3.up * (waterHeight - lastWaterHeight));
}
```

**Что происходит на самом деле:**
1. Каждый кадр считывается высота визуальной воды в позиции игрока
2. Если игрок ниже порога — флаг `swimming = true`
3. Игрок **телепортируется** вверх/вниз на дельту высоты воды (`Move(up * delta)`)
4. Никаких сил, инерции, сопротивления, плавучести

**Почему это выглядит как «плохая левитация»:**
- `CharacterController` — кинематический коллайдер, не реагирует на силы
- Нет инерции: игрок мгновенно следует за водой
- Нет сопротивления воды при движении
- Нет покачивания на волнах (только вертикальный сдвиг)
- Нет разницы между поверхностью и глубиной

### 4. `Boyant` — простой снап для объектов

Есть компонент `Boyant` (опечатка вместо Buoyant) — делает то же самое, но для нефизических объектов:

```csharp
// Boyant.Update()
float y = Ocean.GetWaterHeightAtLocation2(x - choppy, z) + buoyancy;
transform.position = new Vector3(x, y, z);  // прямая установка позиции
```

Используется для поплавков рыбалки и аналогичных объектов. Не для игрока.

## Сравнительная таблица

| Аспект | Корабль | Игрок |
|--------|---------|-------|
| **Физический компонент** | `Rigidbody` (динамика) | `CharacterController` (кинематика) |
| **Взаимодействие с водой** | Сила выталкивания в точках | Прямое позиционирование |
| **Инерция** | Да (масса Rigidbody) | нет (Move мгновенный) |
| **Крен/дифферент** | Да (момент сил) | нет |
| **Покачивание на волнах** | Через силы + демпфирование | Вертикальный телепорт |
| **Сопротивление воды** | Да (drag, dampCoeff) | нет |
| **Реалистичность** | Высокая (иллюзия) | Нулевая |

## Почему так сделано

1. **Производительность:** `CharacterController` дешевле для FPS-контроллера, не требует симуляции
2. **Предсказуемость:** для геймплея важнее контролируемость, чем реализм
3. **OVR-интеграция:** движение построено на `OVRPlayerController`, который работает через CharacterController
4. **Корабли отдельно:** корабли — редкие объекты, можно позволить Rigidbody-физику

## Следствия для мододела

### Если хотите реалистичное плавание

**Вариант A: Добавить Rigidbody-физику игроку (сложно)**
- Заменить `CharacterController` на `Rigidbody` + кастомный контроллер
- Добавить `Buoyancy` или аналог
- Риск: сломать всю систему движения, прыжков, карабканья

**Вариант B: Улучшить иллюзию через CharacterController (реалистично)**
- Добавить инерцию: вместо мгновенного `Move` — лерп к целевой высоте
- Добавить горизонтальное покачивание от `choppy`
- Добавить сопротивление при погружении (замедление Move)
- Добавить волны через `AddForce` на отдельный Rigidbody-пустышку

**Вариант C: Гибрид (рекомендуется)**
- Оставить CharacterController для движения
- Добавить дочерний Rigidbody-объект для визуального покачивания
- Синхронизировать позицию через `Move` с лагом

### Ключевые поля для тюнинга

```csharp
// PlayerSwimming — основные параметры
swimHeight = ...;            // порог глубины для плавания
shallowWaterHeight = 0.6f;   // порог мелководья
surfaceSwimHeight = ...;     // порог поверхностного плавания
diveSpeedMult = ...;         // множитель ныряния
jumpKeyUndiveSpeed = ...;    // скорость всплытия
lerp = ...;                  // скорость следования за водой (чем больше — тем резче)

// Для модификации плавучести — Buoyancy (на корабле)
magnitude = 2f;              // сила выталкивания
dampCoeff = 0.1f;            // демпфирование
CenterOfMassOffset = -1f;    // центр масс
```

### Антипаттерны

- ❌ Не пытайтесь добавить `Rigidbody` на префаб игрока — сломается OVR-движение
- ❌ Не меняйте `CharacterController.isKinematic` — это не то же самое
- ❌ Не используйте `Boyant` на игроке — он перезапишет позицию каждый кадр, конфликтуя с движением
- ✅ Для иллюзии физики — модифицируйте `PlayerSwimming` или добавьте пост-обработку в `LateUpdate`

## Итог

**Воды как физического объёма в Sailwind нет.** Есть:
- Визуальный меш с API высоты (`Ocean`)
- Для кораблей — силы выталкивания на Rigidbody (иллюзия физики)
- Для игрока — прямое позиционирование по высоте поверхности (левитация)

Ваше ощущение «плохой невесомости» абсолютно точно описывает то, что происходит в коде.
