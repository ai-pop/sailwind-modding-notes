# 45. Ящики/грузовые префабы и фильтрация (LootTable)

Разбор `ShipItemCrate` и того, как отличить «грузовой ящик с лутом» от прочих предметов с тем же компонентом. Информация получена декомпиляцией `Assembly-CSharp.dll` (Sailwind v0.38) через ILSpy. Связано с заметками 16 (ShipItem), 32 (инвентарь/груз), 33 (спавн/подбор), 44 (twin).

## `ShipItemCrate` — это универсальный «контейнер/стопка»

`ShipItemCrate : ShipItem` — компонент «внутри N копий другого предмета». Его имеют **не только деревянные грузовые ящики**, но и другие «стопки/упаковки» товаров — например, **`tobacco big`**, **`apples`** и подобные. Поэтому `GetComponent<ShipItemCrate>()` **даёт ложные срабатывания**, если нужен именно грузовой ящик.

### Поля
| Поле | Тип | Содержание |
|------|-----|-----------|
| `containedPrefab` | `GameObject` (`[SerializeField] private`) | Префаб предмета **внутри**. |
| `amount` | `float` (наследуется от `ShipItem`) | **Число предметов внутри**; `> 0` = запечатан, `0` = пуст/распечатан. |
| `smokedFood` | `bool` | Содержимое — копчёная еда (влияет на стоимость/состояние). |
| `goodC` | `Good` (private) | Ссылка на `Good` (если ящик — торговый/миссионный товар). |
| `crateInventory` | `CrateInventory` (private) | Инвентарь ящика (добавляется в `OnLoad`). |
| `crates` | `static Dictionary<int, CrateInventory>` | Реестр ящиков. |

### Семантика `amount` / `containedPrefab`
- `amount > 0` → ящик **запечатан**, внутри `amount` копий `containedPrefab`.
- `amount == 0` → ящик **пуст/распечатан** (содержимое уже в `CrateInventory` или отсутствует).
- `containedPrefab == null` → «странный»/декоративный префаб без реального содержимого (таких лучше избегать в LootTable).
- `OnLoad()` пересчитывает стоимость: `value = containedValue * amount + 20` (× 1.2 если `smokedFood`).

### Вскрытие vs открытие (`OnAltActivate`)
- **Запечатан** (`amount > 0`) и **не миссионный** (`goodC == null || goodC.GetMissionIndex() == -1`) → `CrateSealUI.ShowUI(this)` (подтверждение **вскрытия** → `UnsealCrate`: спавнит `amount` копий содержимого **в инвентарь ящика**, `amount → 0`).
- **Пуст** (`amount <= 0`) и есть `CrateInventory` → `OpenCrate()` (открыть инвентарь ящика).
- **Миссионный** ящик (`goodC.GetMissionIndex() > -1`) свободным вскрытием не открывается (сдаётся через доставку).

## Чем «грузовой ящик» отличается от «стопки товара»

На уровне компонента — **ничем принципиальным**: `ShipItemCrate` один и тот же. Различать нужно по **данным префаба**:

| Критерий | Что даёт |
|----------|----------|
| `containedPrefab != null` | внутри действительно что-то есть (отсекает пустые/битые префабы). |
| `amount > 0` | запечатан (есть лут). |
| Имя префаба / паттерн | «деревянные» грузовые ящики обычно имеют характерные имена (`crate…`); `tobacco big`, `apples` — это «ящик/стопка товара», не грузовой контейнер. |
| `Good` + `TransactionCategory` | ящик-товар (торгуется/возится) vs бытовая стопка. |
| Визуал/масса | грузовой ящик тяжелее и выглядит как контейнер. |

> **Точный список индексов/имён «настоящих» грузовых ящиков из одной декомпиляции не получить** — это данные префабов/сцен. Список собирается **runtime-перечислением** (см. ниже) и фильтрацией по имени/содержимому.

## Runtime-перечисление и фильтр для LootTable

```csharp
// собрать кандидатов один раз (например, в Awake мода, ПОСЛЕ инициализации PrefabsDirectory)
List<GameObject> cargoCrates = new();
foreach (var p in PrefabsDirectory.instance.directory)
{
    if (p == null) continue;
    var crate = p.GetComponent<ShipItemCrate>();
    if (crate == null) continue;

    // базовый фильтр: есть содержимое
    var contained = crate.GetContainedPrefab();      // public-метод
    if (contained == null) continue;

    // отсечь «стопки товаров» (tobacco big, apples и т.п.) — по имени/категории
    string n = p.name.ToLowerInvariant();
    bool isCargoCrate = n.Contains("crate")          // эвристика по имени
                        /* или своя таблица имён/индексов */;
    if (!isCargoCrate) continue;

    cargoCrates.Add(p);
}
// залогируй cargoCrates (имя + prefabIndex + containedPrefab.name + amount),
// чтобы увидеть реальный список и уточнить фильтр под версию игры.
```

- `GetContainedPrefab()` — **публичный** метод `ShipItemCrate` (префаб содержимого).
- `amount` — публичное поле `ShipItem`.
- Имена/индексы префабов для точного фильтра — **логируются в runtime** (декомпиляция их не даёт). Эвристика `name.Contains("crate")` — стартовая точка; уточнить по логу.
- «Пустые»/странные префабы (`containedPrefab == null` или `amount <= 0`) из LootTable исключать.

## Поведение при спавне ящика (что получит игрок)

- Заспавненный ящик с `amount > 0` — **запечатан**; игрок подбирает его **в руки** (крупный предмет не лезет в слот инвентаря, заметка 32), ставит на палубу, ПКМ → вскрытие → содержимое в `CrateInventory` → достать по одному.
- Ящик **не исчезает** при вскрытии — остаётся контейнером (заметка 33).
- Масса ящика = `crate.mass + contained.mass * amount` (`UpdateMass`, заметка 44) — полный ящик тяжёлый.

## Практические выводы

1. `ShipItemCrate` = универсальный «контейнер/стопка N предметов»; `GetComponent<ShipItemCrate>()` **цепляет не только грузовые ящики** (tobacco big, apples и т.п.).
2. `amount` = число предметов внутри (`>0` запечатан); `containedPrefab` = что внутри (`GetContainedPrefab()`); `containedPrefab == null` → пустой/странный префаб.
3. «Настоящий грузовой ящик» отличается **по данным префаба** (имя/содержимое/категория), а не по типу компонента; точный список — runtime-перечислением + фильтром (эвристика `name.Contains("crate")` + лог).
4. Вскрытие (`amount>0`, не миссия) → `CrateSealUI` → содержимое в `CrateInventory`; миссионный ящик свободно не вскрывается.
5. Для LootTable: фильтр `containedPrefab != null` + `amount > 0` + отсечь «стопки товаров» по имени/категории.
