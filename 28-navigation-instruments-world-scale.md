# 28. Навигационные инструменты и масштаб мира

Разбор навигационных приборов и — важно — **вывод о масштабе мировых единиц**. Информация получена декомпиляцией `Assembly-CSharp.dll` (Sailwind v0.38) через ILSpy.

## Масштаб мира: 1 единица = 1 метр

`Speedometer` переводит скорость (`Rigidbody.velocity`, м/с) в единицы измерения:

```csharp
outSpeed = velocity.magnitude * 1.944f;   // → узлы  (1 м/с = 1.944 узла)
outSpeed = velocity.magnitude * 3.6f;      // → км/ч  (1 м/с = 3.6 км/ч)
```

Оба коэффициента соответствуют переводу **из м/с**, значит `velocity` выражен в м/с, а позиции — в **метрах**. **1 мировая единица Unity = 1 метр.**

### Связанные масштабы (сводка по другим заметкам)
| Величина | Формула | Единицы |
|----------|---------|---------|
| Скорость | `velocity` | м/с (× 3.6 = км/ч, × 1.944 = узлы) |
| «Дистанция» миссий | `Vector3.Distance / 100` | **гектометры** (условные «км», заметка 15) |
| Координаты глобуса | `(pos - offset) / 9000` | «градусы» глобуса (заметка 11/17) |
| Игровая зона | globe x[-12,32] z[26,46] | в единицах глобуса (заметка 19) |

> Т.е. мир в метрах, но «км» в миссиях — это 100 м (гектометры), а координаты глобуса получаются делением метров на 9000.

## Компас (`Compass`)

Магнитная стрелка, указывающая на «север»:
- `northAngles` (по умолч. `(-90, 0, 180)`) — ориентация севера (`Quaternion.Euler`).
- `magnetism` (1) — скорость доворота стрелки к северу (`Slerp`, `deltaTime * magnetism`).
- `lockAxis` — маска осей (какие оси стрелки фиксируются/свободны).
- Север — **фиксированное мировое направление** (не зависит от позиции), стрелка плавно наводится на него.

## Лаг / спидометр (`Speedometer`)

- Берёт горизонтальную скорость (`velocity.y = 0`).
- Выводит в `TextMesh` две строки: `"{узлы} knots"` и `"{км/ч} km/h"`.
- `outSpeed` — наружу (для других систем/debug).

## Флюгер (`WindDirectionDisplay`)

Показывает направление **кажущегося ветра**:
- Если задан `shipRigidbody` → `LookAt(pos + (Wind.currentWind - shipRigidbody.velocity))` (ветер минус скорость лодки = кажущийся ветер).
- Иначе → `Wind.windRotation` (истинный ветер).

## Хронометр времени (`ChronometerTime`)

Циферблат, приводимый игровым временем:
```
rotation.z = 180 + Sun.sun.globalTime * 15    // 15° на час
```
Показывает **глобальное** время (`globalTime`, не локальное), 15°/час как у реальных часов.

## Измеритель широты (`ChronometerLatitude`)

Инструмент определения широты по солнцу (типа квадранта/октанта):
- `Scroll(input)` крутит `currentRot` на ±0.5, **кламп `[-45, -11]`** — соответствует диапазону игровых широт.
- `gnomonAligner` (выравниватель по солнцу): поворот `1.475 * currentRot + 30`.
- `latitudeDial` (шкала широты): `num = Angle(forward, parent.up)*2 - 90`; поворот = `latitudeDialRotOffset + 198 + num*2.5`.
- Совместно с солнечным компасом (`GameState.holdingSunCompass`, заметка 18) позволяет определять широту по положению солнца.

## Практические выводы для мододела

1. **1 мировая единица = 1 метр** (вывод из коэффициентов спидометра 3.6 и 1.944). Используйте это для расчётов расстояний/скоростей в модах.
2. **«км» миссий ≠ реальные км**: это `метры / 100` (гектометры). Не путайте при расчёте `pricePerKm` и дедлайнов.
3. **Координаты глобуса** = метры / 9000 (для широтных/долготных эффектов: пассаты, длина дня, выбор рыбы).
4. **Север компаса** — фиксированное направление `Euler(-90,0,180)`; магнитных аномалий нет.
5. **Флюгер** показывает кажущийся ветер (ветер − скорость лодки) — важно для реализации своего ветроуказателя.
6. **Хронометр** идёт по `globalTime` (15°/час), измеритель широты — по солнцу, диапазон широт `[-45,-11]`.
7. Все приборы — `TextMesh`/повороты трансформов; перевод надписей (`"knots"`, `"km/h"`) через подмену текста (заметки 01/08).
